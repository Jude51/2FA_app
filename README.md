# 2FA App — Двухфакторная аутентификация

Это **учебный проект веб-приложения** на Flask с системой регистрации пользователей и **защитой аккаунтов через двухфакторную аутентификацию (2FA)**.


## Как это работает?

### Процесс регистрации
1. Новый пользователь открывает страницу регистрации
2. Вводит **имя пользователя** и **пароль**
3. Выбирает способ двухфакторной аутентификации:
   - **TOTP** — коды генерируются на телефоне в специальном приложении (Google Authenticator, Authy)
   - **Email** — коды отправляются на указанный email
4. Если выбран TOTP, пользователь сканирует **QR-код** в приложении на телефоне
5. Пароль сохраняется в базе данных в **зашифрованном виде** (bcrypt)

### Процесс входа
1. Пользователь открывает форму входа
2. Вводит **имя пользователя** и **пароль**
3. Если пароль верный, система переводит его на экран верификации 2FA:
   - Для TOTP: нужно ввести код из приложения на телефоне
   - Для Email: система отправляет 6-значный код на почту, пользователь вводит его
4. После ввода верного кода пользователь получает доступ в приложение

### Процесс выхода
1. Пользователь нажимает кнопку "Выйти"
2. Сессия очищается, он выходит из аккаунта

## Возможности приложения

✅ **Регистрация новых пользователей** с выбором метода 2FA  
✅ **Безопасное хранение паролей** с помощью bcrypt (криптографическое хеширование)  
✅ **TOTP (Time-based One-Time Password)** — динамические коды через Google Authenticator  
✅ **Email 2FA** — отправка кодов на электронную почту  
✅ **QR-код для TOTP** — удобное сканирование в приложении  
✅ **Управление сессиями** — отслеживание авторизованных пользователей  
✅ **Безопасная конфигурация** — все секреты хранятся в переменных окружения, а не в коде

## Установка

1. Клонируй репозиторий:
```bash
git clone https://github.com/Jude51/2FA_app.git
cd 2FA_app
```

2. Установи зависимости:
```bash
python -m pip install -r requirements.txt
```

3. Создай файл `.env` в корне проекта:
```bash
cp .env.example .env
```

## Настройка

Отредактируй файл `.env` и добавь свои значения:

```env
SECRET_KEY=your-secret-key-here
EMAIL_SENDER=your-email@example.com
EMAIL_PASSWORD=your-email-password
DATABASE_URI=sqlite:///users.db
```

**Важно:**
- `SECRET_KEY` — случайная строка для Flask
- `EMAIL_SENDER` — Gmail адрес для отправки кодов
- `EMAIL_PASSWORD` — пароль приложения Gmail (не главный пароль)

## Запуск

Используй **безопасную версию** приложения:

```bash
python app_safe.py
```

## Как это работает технически

### База данных
Приложение использует **SQLite** — простую встроенную базу данных. В ней хранится информация о пользователях:
- **Имя пользователя** (уникальное)
- **Пароль** (зашифрован с помощью bcrypt)
- **TOTP-секрет** (используется для генерации кодов в Google Authenticator)
- **Email** (для отправки кодов)
- **Способ 2FA** (TOTP или email)

### Хеширование паролей (bcrypt)
Пароль **никогда** не хранится в открытом виде. Вместо этого используется **bcrypt** — криптографический алгоритм:
1. При регистрации пароль преобразуется в хеш (это необратимо)
2. При входе введённый пароль снова хешируется и сравнивается с сохранённым хешем
3. Даже если кто-то получит доступ к БД, пароль не удастся восстановить

### TOTP (Time-based One-Time Password)
Это **алгоритм для генерации одноразовых кодов**:
1. При регистрации приложение генерирует случайный **секретный ключ** (32 символа)
2. Этот ключ кодируется в **QR-код** и передаётся пользователю
3. Пользователь сканирует QR в Google Authenticator, и приложение начинает генерировать 6-значные коды
4. Коды обновляются **каждые 30 секунд**
5. При входе пользователь вводит текущий код, и приложение проверяет его

Это очень безопасно, потому что:
- Коды генерируются **локально на телефоне**, не в интернете
- Никакие коды не отправляются по сети
- Даже если кто-то узнает ключ, коды всё равно будут изменяться

### Email-коды
Для метода email:
1. При входе приложение генерирует **случайный 6-значный код**
2. Отправляет его на email пользователя через **SMTP (Gmail)**
3. Пользователь вводит код в браузер
4. Код проверяется и затем удаляется (одноразовый)

### Переменные окружения (.env)
Вместо того, чтобы писать секреты прямо в коде (опасно!), используются **переменные окружения**:
- `SECRET_KEY` — ключ для Flask (защита сессий)
- `EMAIL_SENDER` — Gmail адрес (отправитель писем)
- `EMAIL_PASSWORD` — пароль Gmail (не главный пароль, а специальный)
- `DATABASE_URI` — путь к БД

При запуске приложение читает эти переменные из файла `.env` (локально) или системных переменных (на сервере).

## Безопасность

- ✅ **Пароли хешируются** с помощью bcrypt — невозможно восстановить исходный пароль
- ✅ **Секреты не в коде** — все ключи и пароли хранятся в `.env`, которого нет в Git
- ✅ **HTTPS для email** — используется защищённое соединение с Gmail (SSL/TLS)
- ✅ **Одноразовые коды** — email-коды используются один раз и удаляются
- ✅ **TOTP в стандарте** — Google Authenticator использует RFC 6238 (признанный стандарт)
- ✅ **Проверка переменных окружения** — приложение отказывается запускаться без необходимых ключей
