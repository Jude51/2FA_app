# 2FA App

Учебный проект с регистрацией, входом и двухфакторной аутентификацией.

## Возможности

- Регистрация пользователей с хешированием паролей (bcrypt)
- Вход с проверкой пароля
- Двухфакторная аутентификация (2FA) двумя способами:
  - **TOTP** (Time-based One-Time Password) через генератор кодов
  - **Email** с отправкой кода на почту
- Безопасное хранение конфигурации через переменные окружения

## Требования

- Python 3.7+
- Flask
- Flask-SQLAlchemy
- bcrypt
- pyotp
- qrcode
- python-dotenv

## Установка

1. Клонируй репозиторий:
```bash
git clone https://github.com/Jude51/2FA_app.git
cd 2FA_app
```

2. Установи зависимости:
```bash
python -m pip install -r requirements.txt
```

3. Создай файл `.env` в корне проекта:
```bash
cp .env.example .env
```

## Настройка

Отредактируй файл `.env` и добавь свои значения:

```env
SECRET_KEY=your-secret-key-here
EMAIL_SENDER=your-email@example.com
EMAIL_PASSWORD=your-email-password
DATABASE_URI=sqlite:///users.db
```

**Важно:**
- `SECRET_KEY` — случайная строка для Flask
- `EMAIL_SENDER` — Gmail адрес для отправки кодов
- `EMAIL_PASSWORD` — пароль приложения Gmail (не главный пароль)

## Запуск

Используй **безопасную версию** приложения:

```bash
python app_safe.py
```

Приложение откроется на `http://127.0.0.1:5000`

## Описание файлов в репозитории

### `app_safe.py`
Основной файл приложения — **безопасная версия**, готовая для использования на GitHub и в production.

**Особенности:**
- Все секреты подгружаются из переменных окружения через `python-dotenv`
- Приложение выбрасывает ошибку при запуске, если отсутствуют необходимые переменные (`SECRET_KEY`, `EMAIL_SENDER`, `EMAIL_PASSWORD`)
- Содержит все маршруты: регистрация, вход, 2FA верификация, выход
- Поддерживает TOTP (Google Authenticator) и email-коды для двухфакторной аутентификации
- Использует SQLite для хранения пользователей и TOTP-секретов
- Пароли хешируются с помощью bcrypt

**Основные маршруты:**
- `/` — главная страница (показывает имя пользователя, если авторизован)
- `/register` — регистрация нового пользователя с выбором метода 2FA
- `/login` — вход по имени и паролю
- `/setup-2fa` — настройка TOTP с QR-кодом
- `/verify-2fa` — ввод кода 2FA (TOTP или email)
- `/logout` — выход

### `requirements.txt`
Список всех зависимостей проекта с точными версиями:
- Flask — веб-фреймворк
- Flask-SQLAlchemy — ORM для работы с БД
- bcrypt — хеширование паролей
- pyotp — генерация и проверка TOTP
- qrcode — создание QR-кодов для TOTP
- python-dotenv — загрузка переменных окружения из `.env`

Установка: `pip install -r requirements.txt`

### `.gitignore`
Правила исключения файлов из Git репозитория. Защищает от случайной загрузки:
- `.env` — конфиденциальные переменные окружения
- `app.py` — локальная версия с жёстко закодированными секретами
- `test_app.py` — файлы тестов
- `instance/users.db` — база данных пользователей
- `__pycache__/` — кеш Python
- `.pytest_cache/` — кеш тестов
- `.coverage` — отчёты покрытия

### `templates/` — HTML-шаблоны

#### `index.html`
Главная страница приложения:
- Если пользователь авторизован — показывает приветствие с его именем и кнопку выхода
- Если не авторизован — предлагает войти или зарегистрироваться

#### `register.html`
Форма регистрации:
- Поле для ввода имени пользователя
- Поле для ввода пароля
- Выбор метода 2FA: TOTP или email
- Поле для ввода email (если выбран email 2FA)

#### `login.html`
Форма входа:
- Поле для ввода имени пользователя
- Поле для ввода пароля
- После успешного входа переводит на верификацию 2FA

#### `setup_2fa.html`
Страница настройки TOTP:
- Показывает QR-код для сканирования в Google Authenticator или другом приложении
- Содержит инструкции для пользователя

#### `verify_2fa.html`
Форма верификации 2FA:
- Для TOTP: поле для ввода 6-значного кода из приложения
- Для email: поле для ввода кода, отправленного на почту

### `static/style.css`
CSS-стили для оформления веб-интерфейса:
- Содержит стили для форм, кнопок, контейнеров
- Обеспечивает удобный и понятный пользовательский интерфейс

## Локальные файлы (не в репозитории)

При разработке и тестировании локально создаются файлы, которые НЕ синхронизируются с GitHub:

- **`.env`** — конфиденциальные переменные окружения (SECRET_KEY, EMAIL_SENDER, EMAIL_PASSWORD)
- **`app.py`** — локальная версия с жёстко закодированными секретами (для локальной разработки)
- **`test_app.py`** — файлы тестов
- **`instance/users.db`** — база данных с пользователями (генерируется при запуске)
- **`static/qrcode.png`** — сгенерированный QR-код для TOTP (создаётся динамически)
- **`__pycache__/`** — кеш Python-файлов
- **`.pytest_cache/`** — кеш тестов
- **`.coverage`** — отчёты покрытия тестов

Все эти файлы добавлены в `.gitignore` и не попадают в репозиторий.

## Безопасность

- ✅ Пароли хешируются с помощью bcrypt
- ✅ Секреты хранятся в `.env`, а не в коде
- ✅ Используется HTTPS для SMTP
- ✅ `app_safe.py` проверяет наличие необходимых переменных окружения при запуске
- ⚠️ Для production используй полноценный сервис аутентификации
